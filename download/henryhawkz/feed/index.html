This is a kind of GRID-EA, the main feature of which is that we use “degrading method” to control risk.

the mql5 version of this expert linked here.

You know, we could hardly find any stable trend from market of recent years. volatility has taken increasingly proportion in the daily price patterns. so, grid-trading should be the most suitable way to trade such a market. however, along with grid-ea, often comes fearful risk. we may face stop-out if we added too many orders. In response to this problem, we figured out a set of “degrading method”. once we’re trapped into adverse trend, we can just make the grid degraded. then we could get better average price at the cost of small loss. and what’s more important is that the grid structure was kept whole.
Besides, we add several indicator filters in this ea to keep the equity moving smoothly. Usage of StableGrid:
TimeFrame： H1 Suitable Symbols：EURUSD, GBPUSD, USDJPY, AUDUSD, USDCAD, NZDUSD, EURJPY, EURGBP, CADJPY
Instruction of the parameters ：
————————- auto parameter ————————- inp_auto_params = auto parameters switch (recommend:true), optimized parameters have been set inside for all symbols.
———————- symbol mode setting ———————- inp_symbol_mode： single_symbol = trade only one symbol of the current chart. multi_symbol_in_serie = all symbols trade in serie mode. when one has already opened some positons, other symbols are locked. multi_symbol_in_parallel = all symbols trade in parallel mode. all symbols are traded simultaneously. inp_symbol_list = symbols list, split by comma ‘,’ *
* ———————– volume parameters ———————– inp_auto_volume = auto volumes switch (recommend false to new user) only after you have run this EA for some time, and have fully learned about it’s general volume, profit and loss in real-time, you may turn this switch on. inp_auto_volume_per_10000 = factor used on auto volumes calculation. value means the lots we trade per 10000 USD AccountBalance. when the calculated result happened to be less than 0.01, EA will set 0.01 as the initial lots. inp_fixed_init_volume = fixed initial lots. set manually.

all the sets above affect only the initial order. the lots of following orders will be multiple of the initial lots.

   **----------------------- risk control parameters -----------------------**   inp\_balance\_percent\_for\_degrade :  when account loss reached a certain percentage of the account balance, EA will launch it's risk control mechanism. that is to partially cut off some positions , we call it degrading. default percentage is 10%. please pay attention to this parameter, if this value is too little, it may cause frequent degrading, account profit will be gone with it. if we set bigger volume on small account, it'll also cause the same problem. so you may moderately magnify this parameter when you are running a small account.

inp_lowest_equity_to_stop_EA : when account equity is less than the value of this parameter, ea stop running. this is the final stop loss of the account.
inp_close_orders_while_stop_EA : switch, whether to close all positions when ea was stopped by account equity limitation.
the parameters above should be set manually. when you run it on auto_param mode, only the grid interval and indicator parameters would be set automatically.
Advice on Account Balance and Order Lots：
balance less than 500 $: use cent account balance 1000 – 3000 $: choose ‘multi_symbol_in_serie’ mode to trade all symbols initial lots = 0.01 balance more than 3000 $: choose ‘multi_symbol_in_parallel’ mode to trade all symbols initial lots = 0.01 balance more than 5000 $: could moderately increase volume. eg. using 0.03 for 10000 usd account.

How to do optimization:
you may optimize parameters yourself. but you need to do some setting first.
1、 inp_auto_params: set to false 2、 inp_symbol_mode: set to single_symbol 3、 inp_auto_volume: set to false 4、 inp_fixed_init_volume: volume should be set smaller. too large a volume will interrupt the optimization. 5、 inp_balance_percent_for_degrade: this param could also be optimized. value range is 5 – 100. 6、 inp_lowest_equity_to_stop_EA this should be set to 0 before optimization. that’s it.
